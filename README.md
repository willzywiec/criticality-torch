# criticality-torch

Modeling fissile material operations in nuclear facilities: a deep-neural-network
metamodel and Bayesian network for estimating process criticality accident risk.

This is a Python/[torch](https://pytorch.org/) port of the R
[`criticality`](https://github.com/willzywiec/criticality) package
(Zywiec et al. 2021, <doi:10.1016/j.ress.2020.107322>). The Monte Carlo
transport engine is [OpenMC](https://openmc.org/): training data is generated by
building spherical fissile-material models and running k-eigenvalue
calculations, replacing the original precomputed MCNP datasets.

## The pipeline, in two stages

1. **Generate** — run a large series of independent OpenMC k-eigenvalue
   calculations *in parallel* (one run per sampled configuration, fanned out
   across a process pool), writing a labeled `keff` output table.
2. **Train** — the neural-network pipeline reads that output, builds a scaled
   train/test dataset, fits an ensemble of DNN metamodels, builds the Bayesian
   network from facility data, and estimates criticality accident risk.

```bash
cd python

# Stage 1: parallel OpenMC generation only -> extdata/openmc.csv
python -m criticality --ext-dir extdata --simulate 16752 --simulate-only --sim-workers 16

# Stage 2: train + estimate risk from that output (no re-simulation)
python -m criticality --ext-dir extdata

# ...or do everything in one command:
python -m criticality --ext-dir extdata --simulate 2000 --device cuda
```

## Getting started

The package lives in [`python/`](python/). OpenMC is not on PyPI, so the
supported setup is a conda-forge environment — on WSL or any Linux, one command
bootstraps everything (Miniforge, the env, cross-section data, and a live
OpenMC verification):

```bash
cd python
bash scripts/setup_wsl.sh
conda activate criticality
```

See [`python/README.md`](python/README.md) for the full reference: the WSL
setup knobs, the Python API, the OpenMC geometry model, and the R → Python
function mapping.
